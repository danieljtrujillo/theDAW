# theDAW (stable-audio-3) — Codebase Audit Synthesis

## Remediation status (updated 2026-07-01)

Fixed so far (verified: tsc / ruff / py_compile / build / .gan compose):
- **Stripped ~8.6MB of dead base64** from `ares/project.json` (7MB→66KB) and `the-owl/project.json` (2MB→22KB). Runtime uses the `.png` file, so no visual change.
- **`set_module_enabled` path traversal** — now validates the resolved path stays a direct child of `backend/modules`.
- **JOBS dict unbounded** — added `_prune_jobs()` (cap 40, evicts oldest finished jobs).
- **`/api/generate` (sync) now holds `_generation_job_lock`** so offload/onload can't move weights mid-generation.
- **liveMixer per-frame `JSON.stringify`** — the editor subscription now gates the FX reconcilers on `tracks`/`masterFxChain` reference changes, so the 60Hz playhead tick no longer re-serializes every chain.
- **Bundle: heavy eager libs split out** — `manualChunks` for `@google/genai`, `react-markdown`/`remark-gfm`/`marked`, `spessasynth_*`; main entry chunk 2,161kB → 1,421kB (−740kB).
- **Deferred first paint via `React.lazy`** — `AssistantPanel` (App.tsx, mount-gated to first orb open), `DocsModal` (Shell.tsx), and the `MidiPanel` MIDI tab (BottomMultiTabPanel, loads on tab open). `triggerPianoNoteFromMidi`/`triggerPianoNote` were extracted into `lib/pianoTrigger.ts` so App + the Sway surface play controller notes without importing the PianoRoll graph. Net: `@google/genai` (276kB) is no longer statically imported by the entry — it now loads only with the MIDI tab — and the entry chunk dropped 1,421kB → ~1,167kB. (verified: tsc + vite build; the deferred panels still opening is pending in-app eyes.)
- **Oversized boot assets re-encoded** — `GANTASMO_LOGO.webp` 2.62MB → 0.90MB (animated, re-encoded to 2x the 110px display cap; all 74 frames preserved), `piz_compressed.exr` 1.79MB → 0.91MB (equirectangular env map downscaled 1024×512 → 512×256, zip16), `theDAW.gltf` 1.57MB → `theDAW.glb` 1.18MB (lossless binary pack of the base64-embedded buffer; loader URL updated). ~3.0MB off the pre-interaction boot payload. (verified: encode integrity + GLB parse structure; the cinematic look/animation is pending in-app eyes.)
- **WaveformEditor 60fps re-render (audit #1)** — the timeline no longer subscribes to `playheadSec`; the moving playhead line/handle + timecode readouts are driven imperatively via refs + a store subscription, so the ~60Hz playback tick stops re-rendering every clip and its per-sample waveform bars. The FX/fader automation overlay keeps following the playhead via a narrow selector that only re-renders during automation-read playback with active lanes. (verified: tsc + vite build; playback smoothness + automation-follow pending in-app ears/eyes.)

Next: oversized `screenshots/` (~17MB PNGs) to WebP + lazy, the remaining eager bottom-panel tabs (Score/alphaTab-OSMD, Draw, StepSequencer) behind `React.lazy`, and the rest below. WaveformEditor could go further (memoized Clip component + canvas/SVG waveform polylines) but the dominant 60fps cost is now removed.

---

theDAW is a text-conditioned audio generation and production suite built on Stable Audio 3, comprising a FastAPI backend (port 8600), a React/Vite frontend (port 5173), an Electron shell, and several Python sidecars (Magenta MRT2, Kinect/akvj, stems separation). This document consolidates the raw findings of 14 parallel subsystem auditors covering frontend state/lib/components/views/build/assets, backend server/media/plugin/integration/AI, Stable Audio 3 inference internals, and the sidecar/Electron layer. Every finding submitted by the auditors is preserved below; only true duplicates were merged (noted inline where applicable). Model-catalog entries are flagged for verification only, never for removal, per project policy.

### Top ~10 highest-impact items

1. **WaveformEditor re-renders the entire timeline (~thousands of DOM nodes) 60x/sec during playback** — the dominant editor frame-time cost; playback jank on any non-trivial project. (Frontend audio components)
2. **liveMixer editor subscription runs N+1 JSON.stringify of all FX chains ~60x/sec during playback** — per-frame CPU + allocation that scales with track count and VST state size. (Frontend state)
3. **`/api/generate` does not take the generation lock** — races `/api/generate-jobs` and defeats offload/onload guards; weights can move out from under an in-flight generation. Correctness bug on the core path. (Backend server)
4. **`set_module_enabled` path traversal** — unvalidated `module_name` in a filesystem write primitive; out-of-tree JSON write. (Backend server)
5. **JOBS dict grows unbounded, storing base64 audio + spectrograms forever** — tens-to-hundreds of MB leaked per long session. (Backend server)
6. **Eager import chains weld genai + spessasynth + three + markdown into the 2.16MB entry chunk** — several hundred KB of unnecessary first-paint JS (App → PianoRoll; App → LoadingScreen → three; App → AssistantPanel). (Frontend build)
7. **~6.1MB of oversized boot assets (2.68MB WEBP logo + 1.84MB EXR + 1.6MB GLTF) fetched before interaction** — the logo re-encode alone is a ~2.6MB win for near-zero effort. (Frontend build / assets)
8. **Library list + aggregate endpoints do full filesystem walks and N+1 DB queries** — every catalogue/stems/MIDI/scores/graph open stalls a worker despite an available SQLite mirror. (Backend integration)
9. **Single analysis pass decodes the same file 4-5x (N+1 decode)** — multiplies auto-analysis cost ~4x on import/idle. (Backend media/FX)
10. **process-mode DSP handlers run heavy synchronous work inside async handlers** — one `/process` call freezes the entire FastAPI event loop for the whole render. (Backend media/FX); paired with the **MRT2 streaming loop having no error handling** (one exception silently kills the audio stream forever). (Sidecars)

## Optimization findings by subsystem

### Frontend state stores & engines

- **liveMixer editor subscription runs JSON.stringify on every FX chain ~60x/sec during playback** — frontend/src/state/liveMixer.ts:616,704-715,209-238 — high/medium — tick()'s per-frame `setPlayhead` fires the editor subscriber, which unconditionally re-stringifies the master chain plus every track's fxChain (including VST base64) as a change-guard; N+1 serializations per frame. Fix: gate the reconcilers on actual masterFxChain/tracks reference changes (zustand supplies prev/next), or move FX reconciliation off the playhead subscription entirely.
- **computePeaks constructs a fresh AudioContext per call** — frontend/src/state/editorStore.ts:870-900 — high/medium — new realtime AudioContext per call (closed in `finally` without awaiting), called from ~15 sites incl. Promise.all bulk loops; hits Chrome's ~6-context cap → blank waveforms + unhandled rejections during import. Fix: use one shared Offline/decode context (or the playerStore engine ctx) and downsample without a live context.
- **surfaceLayoutStore deep-clones the whole node tree AND persists it on every drag mutation** — frontend/src/state/surfaceLayoutStore.ts:113-136,536-570,848-859,484-1015 — high/medium — resize/resizeWidget/setWidgetMargin/setPanelPad each `cloneLayout` the full tree per drag frame, and zustand persist writes the whole partialized layout to localStorage synchronously per set. Fix: shallow-copy only the touched container's fr map for per-pixel drags, and throttle/debounce persist to drag-end.
- **libraryStore.getFiltered() recomputes filter+sort and rebuilds per-entry haystack every render** — frontend/src/state/libraryStore.ts:202-260; LibraryView.tsx:432 — medium/medium — invoked in render body; builds a fresh lowercased haystack per entry and returns a new array each call, so any parent re-render reruns it. Fix: useMemo in the view or precompute a lowercased search index per entry at import/update.
- **mixLiveRack.reconcile JSON.stringify's the full chain on every effectChainStore change** — frontend/src/state/mixLiveRack.ts:71-83,97 — medium/small — full JSON (incl. vst.raw_state) as the no-op guard before the cheaper topoSig check; fires rapidly on knob drags. Fix: build a lightweight param signature or diff per-entry against last-applied values.
- **studioStore.processChain refetches each intermediate stage via fetch(objectURL)+blob()** — frontend/src/state/studioStore.ts:378-383,388-392 — low/small — round-trips the just-produced Blob through the object-URL layer per stage. Fix: return/reuse the Blob directly.
- **editorStore history subscriber calls performance.now() and can setState on every doc mutation** — frontend/src/state/editorStore.ts:844-863 — low/small — coalesce guard makes impact low; long fader rides still churn automationLanes arrays + snapshots. Fix (optional): batch automation writes / exclude automationLanes churn from history granularity.

### Frontend lib (DSP, transforms, clients)

- **Per-frame Vector3.clone() in the lattice morph inner loop** — frontend/src/lib/quantumLattice.ts:606 — medium/small — up to 360 clones/frame (~21,600/s) whenever morphEnergy>0.001 in an otherwise allocation-free renderer. Fix: reuse `this.scratchDir` via copy/normalize/addScaledVector.
- **Arpeggiator soundfont notes scheduled via setTimeout, bypassing the lookahead clock** — frontend/src/lib/arpEngine.ts:392-401 — medium/medium — soundfont path converts the sample-accurate `when` back to a wall-clock setTimeout (throttled ≥4ms, worse backgrounded), smearing swing/quantize while the synth path stays tight. - **Cloud brush creates a radial gradient per particle every frame** — frontend/src/lib/drawEngine.ts:1012-1027 — low/small — up to 64 particles × 18 paths, each `createRadialGradient` + 2 addColorStop per frame. Fix: cache per-radius gradient templates and vary globalAlpha.
- **Organic-brush line buckets rebuilt as a fresh object graph every frame** — frontend/src/lib/drawEngine.ts:934-948 — low/medium — `o.lines = {}` + turtle repopulate per growing offshoot per frame. Fix: reuse a persistent bucket (clear arrays in place) or cache turtle output between frames.
- **Redundant Date.now() calls inside the per-stroke render loop** — frontend/src/lib/drawEngine.ts:896-921 — low/small — dozens-to-hundreds of Date.now() per frame plus three mixed time bases. Fix: read `Date.now()` once per frame and standardize on one time base.
